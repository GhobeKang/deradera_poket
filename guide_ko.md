# 게임 서버 연동 가이드 (Google Drive 활용)

현재 만들어진 HTML 게임은 로컬 브라우저에 데이터를 저장하기 때문에, 다른 사람들과 진행 상황을 실시간으로 공유할 수 없습니다. 대신 구글 드라이브(정확히는 **Google 스프레드시트**와 **Apps Script**)를 무료 서버로 활용하여 팀원들이 같은 게임판을 공유하게 만들 수 있습니다.

관리자(형) 분만 이 세팅을 한 번 해주시면 됩니다!

---

## 1단계 : 데이터를 저장할 스프레드시트 만들기
1. 구글 드라이브(Google Drive)에 접속합니다.
2. `새로 만들기` -> `Google 스프레드시트` -> `빈 스프레드시트`를 클릭합니다.
3. 문서 제목을 적당히 짓습니다. (예: `포켓몬게임_DB`)
4. 열린 스프레드시트 화면의 상단 메뉴에서 **[확장 프로그램] -> [Apps Script]** 를 클릭합니다. (새 창이 열립니다)

## 2단계 : 서버 코드(Apps Script) 붙여넣기
1. 새로 열린 Apps Script 화면의 왼쪽 파일 목록에서 `코드.gs`가 선택되어 있을 겁니다.
2. 오른쪽에 있는 기존 코드를 **모두 지우고**, 서버에서 받아온 **`server.gs` 파일 내용 복사해서 붙여넣습니다**.
3. 디스켓 아이콘(저장) 💾 버튼을 눌러 프로젝트를 저장합니다.

## 3단계 : 서버 배포하기 (가장 중요)
1. Apps Script 화면 상단 오른쪽의 **파란색 [배포]** 버튼을 누르고 **[새 배포]**를 선택합니다.
2. 톱니바퀴 아이콘⚙️(유형 선택)을 눌러 **[웹 앱]**을 선택합니다.
3. 설정값을 다음과 같이 맞춥니다:
   - 설명 : 아무거나 입력 (예: `v1`)
   - 실행하는 사용자 : **나 (관리자 본인 계정)**
   - 액세스 위치 : **모든 사용자** *(이걸 꼭 '모든 사용자'로 해야 다른 사람들도 접속(저장/불러오기)할 수 있습니다)*
4. **[배포]** 버튼을 누릅니다.
5. (처음 배포할 때) '액세스 승인' 창이 뜹니다.
   - [액세스 승인] 버튼 클릭 -> 본인 구글 계정 클릭 -> `고급` 클릭 -> `(안전하지 않음) 프로젝트로 이동` 클릭 -> [허용] 클릭을 진행합니다.
6. 배포가 완료되면 **웹 앱 URL**이 하나 나옵니다! (`https://script.google.com/macros/s/..../exec` 형태)
7. **이 URL을 복사해둡니다.**

## 4단계 : 게임 HTML에 서버 주소 넣기
1. 제가 수정해 드린 `game_sync.html` 파일을 메모장(또는 다른 편집기)으로 엽니다.
2. 파일의 제일 아래쪽을 보면 다음 코드가 있습니다.
   ```javascript
   // ★★★ 관리자님! 여기에 복사한 웹 앱 URL을 붙여넣으세요! ★★★
   const SERVER_URL = '여기에_웹_앱_URL을_붙여넣으세요';
   ```
3. `'여기에_웹_앱_URL...'` 부분(따옴표 안쪽)을 **방금 3단계에서 복사한 주소**로 바꿉니다.
4. 파일을 저장합니다.

---

## 5단계 : 팀원들과 게임하기
1. 이제 이 `game_sync.html` 파일을 카톡방이나 구글 드라이브 등을 통해 팀원들에게 전달합니다.
2. 각자 받은 HTML 파일을 더블클릭해서 엽니다.
3. 관리자가 **[💾 서버에 저장]** 버튼을 누르면 구글 스프레드시트에 저장됩니다!
4. 다른 팀원들이 **[📂 서버에서 불러오기]** 버튼을 누르면 관리자가 저장한 최신 상태를 받아와서 똑같은 화면을 볼 수 있습니다.

> **작은 팁:** 
> 각자가 서로 행동을 하고 "나 끝났어, 저장했어" 라고 말해주면, 다음 사람이 "불러오기"를 누르고 자기 차례를 진행하는 턴제 방식으로 쓰시면 충돌 없이 아주 잘 돌아갑니다!
